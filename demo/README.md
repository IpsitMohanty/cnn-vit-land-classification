# Satellite Land Classification — Demo

Upload a 64×64 satellite tile (or click one of the 6 bundled examples) and get a live agricultural/non-agricultural prediction from the PyTorch CNN trained in [`notebooks/05-pytorch-cnn-classifier.ipynb`](../notebooks/05-pytorch-cnn-classifier.ipynb) (98.6-99.75% accuracy on this task).

**Deliberately not using the CNN-ViT hybrid checkpoint here** — it saves as a 100MB-1.9GB file (see [`results/ANALYSIS.md`](../results/ANALYSIS.md) §3 for why the hybrid isn't worth the extra weight anyway: it doesn't beat the plain CNN on this task). The plain CNN checkpoint bundled here is 78MB.

## Two versions of the same demo

| | `app.py` (deployed) | `app_gradio.py` (local only) |
|---|---|---|
| UI | Streamlit | Gradio |
| Inference | `onnxruntime` on `cnn_model.onnx` | `torch` on `cnn_state_dict.pth` |
| Requirements | [`requirements.txt`](requirements.txt) — small, no torch | [`requirements-gradio.txt`](requirements-gradio.txt) — torch/torchvision, 200MB+ |
| Where it runs | Streamlit Community Cloud, and locally | Locally only |

Both wrap the same trained weights and give identical predictions (verified to within float32 noise — see `export_onnx.py` and `tests/test_demo_streamlit.py::TestParityWithTorchDemo`). `app.py` is the one actually deployed: Hugging Face Spaces' free tier no longer supports compute-backed (Gradio/Docker) Spaces, only Static, so this demo deploys to Streamlit Community Cloud instead, where dropping torch from the inference path keeps installs and cold starts small under that platform's free-tier limits. `app_gradio.py` is kept because it costs nothing to keep and means anyone cloning the repo can run either UI locally.

`cnn_state_dict.pth` (the trained torch weights) remains the source of truth. `cnn_model.onnx` is a derived artifact, committed alongside it, regenerated by `export_onnx.py` whenever the checkpoint changes:

```bash
pip install -r requirements-gradio.txt onnx
python export_onnx.py
```

## Run locally

Streamlit (matches the deployed version):

```bash
python -m venv .venv
.venv\Scripts\activate       # Windows
pip install -r requirements.txt
streamlit run app.py
```

Opens at `http://localhost:8501`.

Gradio (local-only alternative, needs torch):

```bash
python -m venv .venv
.venv\Scripts\activate       # Windows
pip install -r requirements-gradio.txt
python app_gradio.py
```

Opens at `http://127.0.0.1:7860`.

## Deploy to Streamlit Community Cloud

1. Push this repo to GitHub (already done for this project).
2. At [share.streamlit.io](https://share.streamlit.io), create a new app from the `IpsitMohanty/cnn-vit-land-classification` repo, branch `main`, main file path `demo/app.py`.
3. Streamlit Cloud finds `demo/requirements.txt` automatically (it looks alongside the entrypoint file) and installs from that — not the repo-root `requirements.txt`, which is for the training notebooks and is never touched by this deploy.
4. On the free tier, the app sleeps after a period of inactivity and wakes on the next visit (with a short delay) unless kept warm separately.

## Honest limitations

The app states this in its own UI, but worth repeating here: this model was trained on one specific distribution of Sentinel-2 satellite tiles, evenly split between two classes, and scores well *on that distribution*. It has not seen aerial photography, drone imagery, ground-level photos, or tiles from other sensors/resolutions/regions. The bundled example tiles are real held-out samples from the training distribution — they're the fair test of this model. Anything else, including your own uploads, is out-of-distribution and the prediction should not be trusted.
